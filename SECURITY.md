# Política de Seguridad y Reporte de Vulnerabilidades 🔐

Dado que **Sentinelle** maneja la moderación de cuentas de redes sociales y prioriza la privacidad absoluta en el dispositivo (*Local-First*), nos tomamos la seguridad de la información con la máxima seriedad.

## Versiones Soportadas

Actualmente, solo se proporcionan parches de seguridad para la última versión activa en producción:

| Versión | Soportada |
| ------- | --------- |
| >= 1.0.x| Sí        |
| < 1.0.0 | No        |

## Cómo Reportar una Vulnerabilidad

**Por favor, NO abras un Issue público en GitHub para reportar una vulnerabilidad de seguridad.**

Si descubres un fallo de seguridad que ponga en riesgo el aislamiento de datos, el cifrado de Room, los tokens locales de sesión o cualquier vector que comprometa la integridad del usuario, sigue este procedimiento:

1. Envía un correo detallado de manera privada a: **bgaleralop@gmail.com**.
2. Incluye en el mensaje:
    - Descripción técnica precisa del vector de ataque.
    - Pasos detallados o script conceptual (PoC) para reproducir la vulnerabilidad.
    - Versión exacta de Android y de la app Sentinelle donde se ha detectado.

### Nuestro Compromiso de Respuesta
- Acusaremos recibo de tu reporte **a la mayor brevedad**.
- Te mantendremos informado durante el desarrollo del parche de corrección.
- Una vez mitigado el riesgo, se publicará la versión correctiva y se te otorgará el crédito correspondiente en las notas de lanzamiento si así lo deseas.
